Git基础操作

安装与配置Git

首先需要安装Git并配置用户信息：

# 安装Git（根据操作系统选择相应方式） # Windows: 下载Git for Windows # macOS: brew install git # Ubuntu: sudo apt install git # 配置用户信息 git config --global user.name "你的用户名" git config --global user.email "你的邮箱" git config --global core.editor "code --wait" # 设置VS Code为默认编辑器

创建仓库

初始化一个新的Git仓库：

# 创建新文件夹并初始化Git仓库 mkdir my-project cd my-project git init # 或者克隆现有仓库 git clone https://github.com/username/repository.git

保存更改

Git工作流程：工作区 → 暂存区 → 仓库

# 检查当前状态 git status # 添加文件到暂存区 git add filename.txt # 添加特定文件 git add . # 添加所有文件 git add *.js # 添加所有js文件 # 提交更改到仓库 git commit -m "描述性提交信息" # 添加并提交的快捷方式（仅针对已跟踪文件） git commit -am "描述性提交信息"

最佳实践

提交信息应该清晰明了，说明本次提交的目的和内容。使用命令式语气，如"Fix bug"而不是"Fixed bug"。

远程仓库操作

推送文件到远程仓库

将本地更改推送到GitHub远程仓库：

# 添加远程仓库（通常命名为origin） git remote add origin https://github.com/username/repository.git # 查看已配置的远程仓库 git remote -v # 推送到远程仓库 git push -u origin main # 第一次推送时建立跟踪关系 git push # 之后简化推送

克隆远程仓库

获取远程仓库的完整副本：

# 克隆仓库 git clone https://github.com/username/repository.git # 克隆到特定文件夹 git clone https://github.com/username/repository.git my-folder # 克隆特定分支 git clone -b branch-name https://github.com/username/repository.git

拉取更新

获取远程仓库的最新更改：

# 获取远程更新（不会自动合并） git fetch # 获取并合并远程更新 git pull # 相当于 git fetch + git merge git pull origin main # 使用rebase而不是merge git pull --rebase origin main

分支管理

分支操作

Git的分支功能让团队协作更加高效：

# 查看所有分支 git branch # 本地分支 git branch -r # 远程分支 git branch -a # 所有分支 # 创建新分支 git branch new-feature # 切换分支 git checkout new-feature git switch new-feature # 新版本推荐方式 # 创建并切换分支 git checkout -b new-feature git switch -c new-feature # 删除分支 git branch -d branch-name # 安全删除 git branch -D branch-name # 强制删除 # 重命名分支 git branch -m old-name new-name

分支合并

将不同分支的工作成果合并到一起：

# 先切换到要合并到的分支（如main） git checkout main # 合并特性分支 git merge feature-branch # 使用no-ff方式合并（保留分支历史） git merge --no-ff feature-branch # 使用rebase整合分支（创建线性历史） git checkout feature-branch git rebase main

Merge vs Rebase

Merge 保留完整历史，适合公共分支。Rebase 创建线性历史，适合本地分支整理提交记录。

解决合并冲突

当Git无法自动合并更改时，需要手动解决冲突：

# 合并时发生冲突 git merge feature-branch # 查看冲突文件 git status # 手动编辑文件解决冲突 # 文件中的冲突标记： # <<<<<<< HEAD # 当前分支的内容 # ======= # 要合并的分支的内容 # >>>>>>> feature-branch # 解决后标记为已解决 git add resolved-file.txt # 完成合并 git commit

团队协作

Pull Request工作流

GitHub协作的标准流程：

1

创建特性分支

从main分支创建新分支进行开发

git checkout -b feature/new-feature
2

开发并提交更改

在特性分支上完成开发工作

git add . git commit -m "实现新功能"
3

推送到远程

将特性分支推送到GitHub

git push -u origin feature/new-feature
4

创建Pull Request

在GitHub上创建PR请求合并到main分支

5

代码审查与合并

团队成员审查代码后合并到main分支

高级技巧

.gitignore文件

指定不需要版本控制的文件和文件夹：

# 忽略所有.log文件 *.log # 忽略node_modules文件夹 node_modules/ # 忽略.env环境文件 .env # 忽略dist文件夹但保留空文件夹 dist/ !.gitkeep # 忽略所有txt文件除了important.txt *.txt !important.txt

SSH密钥认证

使用SSH密钥避免每次输入密码：

# 生成SSH密钥 ssh-keygen -t ed25519 -C "your_email@example.com" # 或者使用RSA ssh-keygen -t rsa -b 4096 -C "your_email@example.com" # 将公钥添加到GitHub cat ~/.ssh/id_ed25519.pub # 测试连接 ssh -T git@github.com

实用命令

查看提交历史

git log git log --oneline git log --graph git log -p filename

撤销操作

# 撤销暂存的文件 git reset HEAD filename # 撤销工作区的修改 git checkout -- filename # 修改最后一次提交 git commit --amend # 回退到特定提交 git reset --hard commit_hash

储藏更改

# 临时储藏工作进度 git stash # 查看储藏列表 git stash list # 恢复最新储藏 git stash pop # 应用特定储藏 git stash apply stash@{1} # 清除所有储藏 git stash clear

标签管理

# 创建标签 git tag v1.0.0 # 创建带注释的标签 git tag -a v1.0.0 -m "版本1.0.0" # 推送标签到远程 git push origin v1.0.0 # 推送所有标签 git push origin --tags # 删除标签 git tag -d v1.0.0 git push origin --delete v1.0.0
